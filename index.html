MisterWatch monorepo

Deploy each part from the same repo by choosing a different Root Directory per Vercel project.

Root Directory: (leave empty) — repo root This page is deployed as static HTML (/index.html).
Root Directory: admin Admin SPA at / (standalone). API calls go to VITE_API_BASE (set in Vercel → Environment Variables).
  • admin/vercel.json sets VITE_ADMIN_BASE=/ at build time so asset URLs work at the site root.
  • Set VITE_API_BASE to your backend origin, e.g. https://your-api.vercel.app (no trailing slash).
  • Set ALLOWED_ORIGINS on the backend to include your admin URL.
Root Directory: backend Express API + chatbot static bundle (requires build step).
  • Build copies SPAs into backend/public/ (Vercel serves this folder from the CDN). Set VITE_ADMIN_BASE=/admin/ for the admin bundle.
  • Add env vars under Vercel → Settings → Environment Variables (mirror backend/.env.example — never commit .env).
